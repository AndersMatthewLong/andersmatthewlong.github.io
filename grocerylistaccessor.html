Grocery list accessor

Summary

The Grocery List Accessor does all the data operations for grocery lists in the application. I wrote the majority of the code for the current versions (though they are partially based off of earlier accessors written by team member Natalie Ruckman).

Functions

Below is a list of the functions in this accessor. They all perform basic CRUD (Create, Retrieve, Update, and Destroy) operations on the GrceryList table in the database.

  1. Get: This function retrieves a single grocery list record from the database given its unique id.
  2. Add: This function inserts a given instance of the GroceryList class into the database.
  3. GetAll: This function retrieves all the GroceryList records from the database.
  4. Edit: This function, given the id of an existing grocery list and an edited instance of the GroceryList class, applies all the properties of the edited instance to the existing record.

Finally, the highlight of this file, the Delete function

The Delete function does what it sounds like it does, it deletes a grcoery list record, given its id, however, in order to avoid leaving "orphaned" records of the grocery items on the list, we first cycle through those items, deleting them all.
