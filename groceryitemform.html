Grocery item form

Summary

The Grocery Item Form is the frontend page (View) in which the customer creates new and edits existing grocery items.

Workflow

Below is a summary of the flow of data through the application, and how it works with this form

  1. First, somewhere on the GroceryItem index page, an actionlink is clicked, either the New link or the Edit link. These links redirect to the GroceryItemController.cs class.
  2. From these classes, an instance of the GroceryItemFormViewModel.cs class is made, containing the id of the grocery list the user is viewing, as well as an empty GroceryItem we will be creating / editing.
  3. After this, we are directed to this form. The user is provided with text fields for the grocery item name and quantity.
  4. A checkbox is also available (this is a contribution by team member Natalie Ruckman).
  5. The rest of the form is in the next section.

Finally, the highlight of this file, the hidden fields

While we want the user to input some fields in the form, there are some that they do not want / should not be able to edit. These include the primary key in the database, as well as the foreign key which associates it with the correct list. In this case, we want to implement hidden fields. The primary key was easy to implement, but the foreign key had to be packaged with the GroceryItemFormViewModel to ensure that the form had the correct list to associate with the item.
